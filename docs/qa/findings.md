- [ ] **Demote signal cyan everywhere except the canvas ‚Äî "only the shader is colored"** ‚Äî `#9ee7ff` (`--primary`/`--accent`/`--ring`/`--signal`) appears in the solid Download PNG button, active Compare tint, status dot glow, every slider knob/fill, segmented active state, and the "try a sample" link, directly competing with the shader preview. - [ ] **Hero headline is clipped at the top viewport edge** ‚Äî On desktop (1280) and mobile (390) the display headline "WebGL shaders." has its cap-height/ascenders sheared off by the page top edge; reads as broken, not effortless. No breathing room above the H1. ‚Äî **Fix:** Add top padding to the hero/main container (`pt-16`‚Äì`pt-24` / `clamp(4rem,10vh,8rem)`); set `leading-[1.05]` with explicit `padding-top` (not negative margin) so ascenders aren't cropped. Verify ‚â•48px above the cap line at 1280w and ‚â•24px at 390w. ‚Äî hero section in the page wrapper above Studio (not `studio.tsx`)

- [ ] **Export can produce a blank/partial PNG ‚Äî readiness gate is time-based, not content-based** ‚Äî The tick loop only checks canvas size, waits a fixed `SETTLE_MS=360ms`, then reads. For image shaders the source must be fetched + uploaded to a GL texture asynchronously; large uploads routinely exceed 360ms, so the read captures an empty/half-drawn frame. The off-screen container is `opacity:0`, so there is no visual signal the frame was blank. ‚Äî **Fix:** Gate on real readiness: preload + `img.decode()` before mounting the export instance; after `setFrame(0)` force ‚â•2 rAF frames and verify non-empty output via a tiny scratch pixel readback before `toBlob`; make SETTLE adaptive with the 4000ms ceiling as a hard timeout calling `onDone(false)`. ‚Äî `components/studio/export-renderer.tsx:44-93`

- [ ] **Body text at reduced opacity fails WCAG AA contrast** ‚Äî On card bg `#131720`: `text-muted-foreground/70` (section titles "Adjust"/"Color", category/group hints) = 3.48:1; `text-muted-foreground/60` (image WxH dimensions, per-tab shader count) = 2.89:1. All are 10‚Äì11px normal text and fail 4.5:1. Base `muted-foreground` `#8a93a2` alone is fine (5.79:1). ‚Äî **Fix:** Drop the `/70` and `/60` modifiers and use the full `muted-foreground` token, or add a dimmer token that still clears 4.5:1 (~`#9aa3b2`). ‚Äî `components/studio/shader-picker.tsx:35,41`; `components/studio/control-panel.tsx:131`; `components/studio/studio.tsx:177`

- [ ] **Color & palette swatch hit targets are 24px (need 44px)** ‚Äî The color control wraps `<input type=color>` in a `size-6` (24px) box; palette swatches are `size-6` packed at `gap-1.5`, adjacent and mis-tap-prone on touch (full-width rail on mobile). Below WCAG 2.5.5. ‚Äî **Fix:** Make each interactive swatch ‚â•44√ó44 (or add invisible padding so the touch target reaches 44px while the visual swatch stays small); increase palette gap/per-swatch target; ensure the `<input type=color>` itself fills the target, not just its wrapper. ‚Äî `components/studio/param-control.tsx:107-115,132-145`

- [ ] **Range slider focus is invisible and reset is mouse-only** ‚Äî Native `<input type=range>` is arrow-key operable, but `focus-visible:ring-2` sits on a 4px (`h-1`) track so the ring is a near-invisible sliver, and the thumb gets no custom focus treatment. Double-click-to-reset (`onDoubleClick`) is pointer-only ‚Äî no keyboard path to reset. ‚Äî **Fix:** Enlarge the focusable hit area (taller padded wrapper / min thumb size), add a visible thumb focus ring via `::-webkit-slider-thumb:focus-visible`, and expose reset to keyboard (Delete/Backspace handler or a small reset button). ‚Äî `components/studio/param-control.tsx:28-39`

## P1 ‚Äî Should fix (polish gaps, missing feedback, real correctness/leak risks)

- [ ] **Compare divider has no seam line and the after-label is a joined component name** ‚Äî The handle renders as a lone glowing cyan dot with no divider/grip, reading as a smudge; the after-label shows "IMAGEDITHERING" because it passes `shader.component` (a Pascal/joined identifier) rather than a human title ‚Äî an AI/template tell. - [ ] **`parseFloat` on the range input can commit NaN** ‚Äî `onChange={(e)=>onChange(parseFloat(e.target.value))}`; an empty/non-numeric value yields NaN, stored and passed as a NaN uniform that breaks the render silently ("NaN" shows in the display). ‚Äî **Fix:** `const n = parseFloat(e.target.value); if (Number.isFinite(n)) onChange(n);` (else fall back to `param.default`); apply the same guard wherever numeric uniforms come from inputs. ‚Äî `components/studio/param-control.tsx:36`
